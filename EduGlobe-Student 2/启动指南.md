## 🚀 快速启动（推荐）

### 方法一：一键启动所有服务（最简单）

1. **右键点击** `start-all.ps1` 文件
2. 选择 **"使用 PowerShell 运行"**
3. 等待所有服务启动完成
4. 在浏览器中打开 http://localhost:5173

就这么简单！✨

### 方法二：分步启动

如果需要更多控制，可以分别启动各个服务：

```powershell
# 1. 启动 MySQL 数据库
.\start-mysql.ps1

# 2. 启动后端服务（在 backend/server 目录）
cd backend/server
npm start

# 3. 启动前端服务（在 frontend 目录）
cd frontend
npm run dev
```

### 停止服务

```powershell
# 一键停止所有服务
.\stop-all.ps1

# 或单独停止 MySQL
.\stop-mysql.ps1
```

## 📖 详细启动步骤

### 第一步：MySQL 数据库

#### 自动启动（推荐）

```powershell
.\start-mysql.ps1
```

#### 手动启动

```powershell
# 1. 打开 PowerShell
# 2. 进入 MySQL bin 目录
cd D:\mysql-8.0.23-winx64\bin

# 3. 启动 MySQL（新窗口会打开，不要关闭）
.\mysqld.exe --console
```

#### 验证 MySQL 是否启动

```powershell
# 检查 3306 端口是否监听
netstat -ano | findstr :3306

# 使用 MySQL 客户端测试连接
D:\mysql-8.0.23-winx64\bin\mysql.exe -u root -p"CHENhaoyu0608.." -e "SHOW DATABASES;"
```

#### 使用 HeidiSQL 连接

1. 打开 HeidiSQL
2. 创建新会话（或使用现有会话）
3. 配置连接信息：
   - **网络类型**: MariaDB or MySQL (TCP/IP)
   - **主机名/IP**: 127.0.0.1
   - **用户**: root
   - **密码**: CHENhaoyu0608..
   - **端口**: 3306
   - **数据库**: eduglobe_db
4. 点击"打开"连接

### 第二步：后端服务

#### 启动后端

```powershell
# 进入后端目录
cd backend/server

# 方式1: 使用编译后的代码启动（推荐，更快）
npm start

# 方式2: 开发模式（需要 src 源代码）
npm run dev
```

#### 验证后端是否启动

后端成功启动后会看到：

```
info: 数据库连接池创建成功
info: 数据库连接成功
info: 🚀 服务器运行在端口 3000
info: 📚 环境: development
info: 🌐 API地址: http://localhost:3000/api
```

#### 测试后端 API

在浏览器中访问：
- http://localhost:3000 （健康检查）
- http://localhost:3000/api （API 根路径）

### 第三步：前端服务

#### 启动前端

```powershell
# 进入前端目录
cd frontend

# 启动开发服务器
npm run dev
```

#### 验证前端是否启动

前端成功启动后会看到：

```
VITE v4.5.14  ready in 10015 ms

➜  Local:   http://localhost:5173/
➜  Network: http://192.168.x.x:5173/
```

#### 访问应用

在浏览器中打开：http://localhost:5173/

## ❓ 常见问题

### 1. MySQL 启动失败

**问题**: 运行 start-mysql.ps1 后 MySQL 无法启动

**解决方案**:
```powershell
# 1. 检查 MySQL 路径是否正确
Test-Path "D:\mysql-8.0.23-winx64\bin\mysqld.exe"

# 2. 检查 3306 端口是否被占用
netstat -ano | findstr :3306

# 3. 如果端口被占用，停止占用进程
# 找到 PID 后：
Stop-Process -Id <PID> -Force

# 4. 如果是 phpStudy 的 MySQL，确保已停止
Get-Process | Where-Object { $_.Path -like "*phpStudy*mysql*" } | Stop-Process -Force
```

### 2. 后端启动失败 - 找不到 src/index.ts

**问题**: `Error: Cannot find module './index.ts'`

**解决方案**:
```powershell
# 使用 npm start（使用编译后的代码）而不是 npm run dev
cd backend/server
npm start
```

### 3. 后端无法连接数据库

**问题**: `Access denied for user 'root'@'localhost'`

**解决方案**:
```powershell
# 1. 检查 .env 文件中的密码是否正确
# 2. 测试 MySQL 连接
D:\mysql-8.0.23-winx64\bin\mysql.exe -u root -p"CHENhaoyu0608.." -e "SHOW DATABASES;"

# 3. 如果密码不对，在 MySQL 中重置密码
# 以无密码方式启动 MySQL，然后：
# ALTER USER 'root'@'localhost' IDENTIFIED BY 'CHENhaoyu0608..';
```

### 4. 前端无法访问后端 API

**问题**: 前端显示网络错误或 CORS 错误

**解决方案**:
```powershell
# 1. 确认后端正在运行
netstat -ano | findstr :3000

# 2. 检查后端 .env 中的 CORS 配置
# CORS_ORIGIN=http://localhost:5173

# 3. 重启后端服务
```

### 5. 端口被占用

**问题**: 端口 3000 或 5173 已被占用

**解决方案**:
```powershell
# 查找占用端口的进程
netstat -ano | findstr :3000
netstat -ano | findstr :5173

# 停止进程（替换 <PID> 为实际进程 ID）
Stop-Process -Id <PID> -Force
```

### 6. PowerShell 脚本无法运行

**问题**: "无法加载文件，因为在此系统上禁止运行脚本"

**解决方案**:
```powershell
# 以管理员身份运行 PowerShell，执行：
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

# 然后重新运行脚本
```

## 💡 开发建议

### 启动顺序

最佳实践是按以下顺序启动：

1. **MySQL** （数据层）
2. **后端** （业务逻辑层）
3. **前端** （展示层）

停止时建议反向顺序。

### 保持窗口打开

启动服务后，不要关闭弹出的命令行窗口，它们会显示：
- 后端日志（API 调用、错误信息）
- 前端日志（页面访问、热更新）
- MySQL 日志（数据库操作）

### 开发模式

前端支持热更新（HMR），修改代码后自动刷新浏览器。

后端目前使用编译后的代码运行，如需修改后端代码：
1. 修改源代码
2. 重新编译：`npm run build`
3. 重启服务：`npm start`
