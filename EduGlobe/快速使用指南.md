# 🚀 EduGlobe 快速使用指南

## 📌 一分钟快速启动

### 方式一：一键启动（推荐）
```powershell
# 在 EduGlobe 目录下运行
.\start-all.ps1
```

### 方式二：手动启动
```powershell
# 1. 初始化数据库（首次运行）
cd backend\database
.\migrate-database.ps1

# 2. 启动后端（在新终端）
cd backend\server
npm install     # 首次需要
npm run dev

# 3. 启动前端（在新终端）
cd frontend
npm install     # 首次需要
npm run dev
```

### 停止服务
```powershell
.\stop-all.ps1
```

## 📖 API调用示例

### 前端调用（推荐方式）
```typescript
import { 
  geopoliticsApi, 
  cyberLabApi, 
  scenarioApi, 
  aiCompanionApi
```

## ⚠️ 常见问题

### 1. MySQL启动失败
**问题**: 提示"MySQL启动失败"
**解决**: 
- 检查`start-all.ps1`中的`MYSQL_PATH`是否正确
- 确保MySQL服务未被占用
- 手动运行`start-mysql.ps1`查看详细错误

### 2. 端口被占用
**问题**: 提示"端口3000或5173已被占用"
**解决**:
```powershell
# 查看占用端口的进程
netstat -ano | findstr "3000"
netstat -ano | findstr "5173"

# 杀死进程（PID为查询结果最后一列）
taskkill /PID <进程ID> /F
```

### 3. 数据库连接失败
**问题**: 后端无法连接数据库
**解决**:
- 检查`backend/server/.env`中的数据库配置
- 确认MySQL已启动
- 确认数据库`eduglobe_db`已创建

### 4. 前端API调用失败
**问题**: 前端无法调用后端API
**解决**:
- 检查`frontend/.env.development`中的`VITE_API_URL`
- 确认后端服务已启动
- 打开浏览器开发者工具查看具体错误

### 5. 依赖安装失败
**问题**: npm install报错
**解决**:
```powershell
# 清除缓存
npm cache clean --force

# 删除node_modules和package-lock.json
rm -r node_modules
rm package-lock.json

# 重新安装
npm install
```
