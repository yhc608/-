## 🚀 快速启动

### 一键启动（推荐）

在项目根目录执行：
```bash
.\start-all.ps1
```

此脚本将自动：
1. 检查并启动 MySQL 服务
2. 初始化数据库（如果需要）
3. 启动后端服务
4. 启动前端服务

### 手动启动

#### 1. 启动数据库
```bash
.\start-mysql.ps1
```

或确保 MySQL 服务已运行：
```powershell
Get-Service MySQL* | Start-Service
```

#### 2. 初始化数据库（首次运行）
```bash
cd backend\database
.\migrate-database.ps1
```

#### 3. 启动后端服务
```bash
cd backend\server
npm install  # 首次运行需要
npm run dev
```

后端将运行在：http://localhost:3000

#### 4. 启动前端服务
```bash
cd frontend
npm install  # 首次运行需要
npm run dev
```

前端将运行在：http://localhost:5173

## 🔧 技术栈

### 后端
- **框架**: Express.js + TypeScript
- **数据库**: MySQL 8.0
- **认证**: JWT + bcryptjs
- **实时通信**: Socket.io
- **API**: RESTful

### 前端
- **框架**: Vue 3 + TypeScript
- **构建工具**: Vite
- **UI**: 自定义组件
- **路由**: Vue Router
- **状态管理**: Composition API
- **HTTP**: Axios

## 🐛 常见问题

### 1. 数据库连接失败
- 检查 MySQL 服务是否启动：`Get-Service MySQL*`
- 确认密码正确：`CHENhaoyu0608..`
- 检查端口 3306 是否被占用

### 2. 后端启动失败
- 确认依赖已安装：`npm install`
- 检查端口 3000 是否被占用
- 查看日志：`backend/server/logs`

### 3. 前端启动失败
- 确认依赖已安装：`npm install`
- 检查端口 5173 是否被占用
- 清除缓存：`npm run build --force`

### 4. API请求失败
- 确认后端已启动
- 检查 CORS 配置
- 查看浏览器 Network 面板

## 📝 开发指南

### 添加新功能模块
1. 在 `backend/database` 创建新的SQL表
2. 在 `backend/server/src` 创建对应的路由、控制器、服务
3. 在 `frontend/src/services/api.ts` 添加API调用方法
4. 在 `frontend/src/pages` 创建页面组件
5. 在 `frontend/src/router` 注册路由

### 代码规范
- 使用 TypeScript 严格模式
- 遵循 ESLint 规则
- 使用语义化命名
- 编写注释说明
